Installing elasticagent and adding a fleet (parent) server

Ideally, you want the server who has kibana that becomes the fleet (parent) server for easier data access through UI. So, in this case I‚Äôll be using http://xxx.xxx.133.123 as the parent server because it has Kibana installed. I would recommend exploring Kibana on your own to see its full capability.

Anyway, to add a fleet server, go to Fleet menu.

Download, extract, and run elastic-agent as fleet server (See command below).

I use the Quick Start option here. If you want to deploy this to production, you should consider using your own certificate.
For ‚ÄúFleet Server hosts‚Äù, this ideally should be the server who has Kibana installed. This server will be the parent server. Fleet server use port 8220 by default. Then, click ‚ÄúGenerate Service Token‚Äù.

Lastly, I literally copy-pasted the command given by the instruction on the Fleet menu.

sudo ./elastic-agent install --fleet-server-es=http://localhost:9200 --fleet-server-service-token=your-token --fleet-server-policy=policy-id --fleet-server-insecure-http

check if parent server successfully added

Next, we want to make a new policy for our child server which we‚Äôll setup in a second.

Go to your newly created policy. Click Actions -> Add Agent.
You‚Äôll see a set of instruction and command. Continue to Setting up child node section.
