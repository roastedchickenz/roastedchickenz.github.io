So I need to find a new way to bypass the root detection. Finally, I use Frida to bypass the root detection mechanism. After starting Frida on my Android emulator, always checks that it has successfully connected to host machine using:

We need to dig through the native function this time. First, we need to find the file of course! It should be on the default location but just in case, we can always use something like

1

Where ‚Äúfoo‚Äù is the native library we want to find as shown by MainActivity.java.
We should see that it‚Äôs located on ./resources/lib/x86_64/libfoo.so

The architecture doesn‚Äôt really matter in this case, I pick x86_64 not for any spesific reason.

When decompiling binaries, I like to start with the easiest thing, giving the binaries to strings command.

1

In this case, the strings command doesn‚Äôt give any useful information (Unless you already know the solution, it gives truncated version of the secret string). Next, try ghidra.

After auto-analyzing with ghidra, the symbol tree will show us existing function

Reading the decompiled function Java_sg_vantagepoint_uncrackable2_CodeCheck_bar , we could find that it is indeed doing a code check. It uses strncmp function.

Which means we should be able to retrieve the compared strings.

On line 31, where it does the strncmp function, it points to the value of local_30.

The value of local_30 is a hex as shown on line 21. In fact, the rest of the secret is laid here in Little-endian and then reversed.
